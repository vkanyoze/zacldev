Welcome to the Payment API documentation. This API allows you to process payments, manage payment methods, and handle transactions programmatically.

Getting Started

To get started with the API, you'll need to obtain an API key by registering an account. All API requests must include your API key in the Authorization header.

Base URL

https://api.yourdomain.com/v1

Rate Limiting

API requests are limited to 60 requests per minute per API key. If you exceed this limit, you'll receive a 429 Too Many Requests response.

Response Format

All API responses are returned in JSON format with the following structure:

{
    "success": true,
    "data": {
        // Response data
    },
    "message": "Operation completed successfully"
}

Authentication

All API requests must include an API key in the Authorization header:

Authorization: Bearer your_api_key_here

Obtaining an API Key

To obtain an API key, register for an account at /register and navigate to the API settings in your dashboard.

Example Request

curl -X GET https://api.yourdomain.com/v1/user \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Accept: application/json"

API Endpoints

Payments

GET /v1/payments Requires Auth

List all payments

Retrieve a paginated list of payments for the authenticated user.

Query Parameters
Parameter Type Required Description
page integer No Page number (default: 1)
limit integer No Items per page (default: 10, max: 100)
Example Response
{
    "success": true,
    "data": {
        "items": [
            {
                "id": "pay_abc123",
                "amount": 1000,
                "currency": "USD",
                "status": "succeeded",
                "created_at": "2023-01-01T12:00:00Z"
            }
        ],
        "pagination": {
            "total": 1,
            "page": 1,
            "limit": 10,
            "total_pages": 1
        }
    }
}
POST /v1/payments Requires Auth

Create a payment

Process a new payment with the provided details.

Request Body
Parameter Type Required Description
amount integer Yes Amount in the smallest currency unit (e.g., cents)
currency string Yes 3-letter ISO currency code (e.g., USD)
description string No Description of the payment
card_id string No ID of a saved card to use (if not provided, will use default card)
Example Request
{
    "amount": 1000,
    "currency": "USD",
    "description": "Monthly subscription"
}
Example Response
{
    "success": true,
    "data": {
        "id": "pay_abc123",
        "amount": 1000,
        "currency": "USD",
        "status": "succeeded",
        "created_at": "2023-01-01T12:00:00Z"
    }
}

Cards

GET /v1/cards Requires Auth

List saved cards

Retrieve a list of saved payment methods for the authenticated user.

Example Response
{
    "success": true,
    "data": [
        {
            "id": "card_abc123",
            "brand": "visa",
            "last4": "4242",
            "exp_month": 12,
            "exp_year": 2025,
            "is_default": true
        }
    ]
}

Error Handling

Errors follow a standard format with an error code and message:

Error Response Format

{
    "success": false,
    "error": {
        "code": "error_code",
        "message": "Human-readable error message"
    }
}

Common Error Codes

Status Code Error Code Description
400 invalid_request The request was invalid or cannot be processed
401 unauthorized Authentication failed or not provided
403 forbidden Insufficient permissions
404 not_found The requested resource was not found
429 rate_limit_exceeded Too many requests
500 server_error An internal server error occurred
